# AI Dispute Resolution Engine

## Project Overview
This repository showcases QA/SDET work validating an **AI-driven dispute management chatbot**
for a fintech platform — covering conversational AI testing, automated dispute workflows, and
anomaly-detection validation. It is a portfolio project built with dummy/sample data only — no
real client names, production data, or confidential information are included.

## My Role
QA Engineer / SDET — responsible for functional and conversational AI testing, dispute workflow
validation, anomaly-detection test design, and defect tracking for this project.

## Tech Stack & Tools Used
Playwright, TypeScript, Postman, JIRA, Git

## Types of Testing Performed
- AI Chatbot Testing (intent recognition, response accuracy, fallback handling, context retention)
- Conversational Flow Testing
- Dispute Workflow Validation (initiation, AI-suggested resolutions, escalation, status updates)
- AI Anomaly Detection Testing (edge-case transaction scenarios, fraud alert thresholds)
- Negative & Boundary Testing on AI Responses (graceful degradation, human-agent fallback)
- API Testing
- Regression Testing

## Key Achievements
- Validated intent recognition, response accuracy, and context retention across dispute
  management conversational flows
- Validated AI-driven dispute flows end-to-end: automated dispute initiation, AI-suggested
  resolutions, escalation triggers, and real-time status updates
- Tested anomaly-detection models by injecting edge-case transaction scenarios to verify fraud
  alerts and threshold breach behavior
- Conducted negative/boundary testing on AI responses to confirm graceful degradation and
  reliable fallback to human agents
- Collaborated on defining acceptance criteria and validating model outputs against quality
  standards prior to release
